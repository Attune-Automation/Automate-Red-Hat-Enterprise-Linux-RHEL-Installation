The following steps will guide you through the manual process.

Step 1 - Clean Build Files

Deletes the build files and kickstart ISO.

Connect via ssh:

ssh {buildserveruser}@{attuneosbuildserver}

Execute the following script: 

                
DIR="{ksAttuneBaseDir}/build-{targetServer.fqn}"
if [ -d "${DIR}" ]
then
    rm -rf "${DIR}" && echo "Removed ${DIR}"
else
    echo "No directory to remove."
fi

ISO="{ksAttuneBaseDir}/kickstart_{targetServer.fqn}.iso"
if [ -f "${ISO}" ]
then
    rm -fv "${ISO}"
else
    echo "No ISO to remove."
fi

Step 2 - Deploy RHEL9 Boot ISO

Deploys a RHEL9 boot ISO.

This file can be downloaded from the following link. You need a Redhat login. https://developers.redhat.com/products/rhel/download#assembly-field-downloads-page-content-61451

Copy the File(s) RHEL_Boot_ISO.unknown to the target node.

Deploy the File(s) here:

{ksAttuneBaseDir}/build-{targetServer.fqn}, relative to the home directory

Step 3 - Extract RHEL ISO

Extracts the boot ISO.

Execute the following script: 

                
cd {ksAttuneBaseDir}/build-{targetServer.fqn}

if ls | grep -q rhel*iso
then
    
    ISO="$(echo rhel*iso)"
    echo "ISO is $ISO"
    
    7z x "${ISO}"
    
    rm -v "${ISO}"
else
    echo "ISO is already extracted."
fi

Step 4 - Deploy RHEL9 UEFI Kickstart Config

RHEL9 UEFI boot files for kickstarting RHEL9.

Copy the File(s) RHEL_UEFI_Kickstart_Config.tar to the target node.

{ksAttuneBaseDir}/build-{targetServer.fqn}, relative to the home directory

Step 5 - Copy RPM CA Certificates from Drop In Directory

Copies any RPM CA Certificates from the drop in directory {rpmCaCertificateDropInDirectory to {ksAttuneBaseDir}/build-{targetServer.fqn}/cacerts.

Execute the following script: 

                
if [ ! -d "{rpmCaCertificateDropInDirectory}" ]
then
    echo "{rpmCaCertificateDropInDirectory} does not exist, skipping."
else
    cd "{rpmCaCertificateDropInDirectory}"
    
    NUMBER_FILES=$(ls -1q | wc -l)
    echo "NUMBER_FILES=${NUMBER_FILES}"
    
    if [ "${NUMBER_FILES}" -eq "0" ]
    then
        echo "No files in {rpmCaCertificateDropInDirectory}, skipping."
    else
        echo "{rpmCaCertificateDropInDirectory} exists, copying it's contents to {ksAttuneBaseDir}/build-{targetServer.fqn}/cacerts."
        cp --verbose --recursive ./* "{ksAttuneBaseDir}/build-{targetServer.fqn}/cacerts"
    fi
fi

Step 6 - RHEL9 Modify EFI/BOOT/grup.cfg in images/efiboot.img

Step 6.1 - Create Temporary Directory for efiboot.img Mount

Creates temporary directory temporary_mount for mounting efiboot.img.

Execute the following script: 

                
cd {ksAttuneBaseDir}/build-{targetServer.fqn}
mkdir temporary_mount

Step 6.2 - Mount grub.cfg inside efiboot.img

The unique directory {ksAttuneBaseDir}/build-{targetServer.fqn}/temporary_mount is a temporary mount for images/efiboot.img.

The decision to mount to a unique directory instead of /mnt is so that multiple kickstarts can be run simultaneously.

Connect via ssh:

ssh {linuxrootuser}@{attuneosbuildserver}

Execute the following script: 

                
cd {ksAttuneBaseDir}/build-{targetServer.fqn}

mount --options loop,gid={buildServerUser.user},uid={buildServerUser.user} ./images/efiboot.img ./temporary_mount

ls -lh ./temporary_mount/EFI/BOOT/grub.cfg

echo "Here are the contents of {ksAttuneBaseDir}/build-{targetServer.fqn}/temporary_mount/EFI/BOOT/grub.cfg"
cat ./temporary_mount/EFI/BOOT/grub.cfg

Step 6.3 - RHEL9 Deploy grub.cfg inside efiboot.img

Deploys RHEL9 grub.cfg to the mounted efiboot.img in folder temporary_mount under EFI/BOOT.

Connect via ssh:

ssh {buildserveruser}@{attuneosbuildserver}

A modified "EFI/BOOT/grub.cfg" suitable for RHEL9 UEFI boot to be put in "images/efiboot.img".

Copy the File(s) RHEL_grubcfg_Inside_efibootimg.tar to the target node.

Extract and deploy the File(s) here:

{ksAttuneBaseDir}/build-{targetServer.fqn}/temporary_mount/EFI/BOOT, relative to the home directory

Step 6.4 - Unmount Temporary Directory for efiboot.img Mount

Unmounts temporary_mount. This writes back the files into images/efiboot.img.

Connect via ssh:

ssh {linuxrootuser}@{attuneosbuildserver}

Execute the following script: 

                
cd {ksAttuneBaseDir}/build-{targetServer.fqn}
umount ./temporary_mount

Step 6.5 - Delete Temporary Directory for efiboot.img

Deletes the temporary directory temporary_mount.

Connect via ssh:

ssh {buildserveruser}@{attuneosbuildserver}

Execute the following script: 

                
cd {ksAttuneBaseDir}/build-{targetServer.fqn}
rm -rvf temporary_mount

Step 7 - Create UEFI Boot ISO

Creates a UEFI bootable ISO containing the kickstart files.

Execute the following script: 

                
cd {ksAttuneBaseDir}/build-{targetServer.fqn}
  
mkisofs -o {ksAttuneBaseDir}/kickstart_{targetServer.fqn}.iso \
  -joliet-long -b isolinux/isolinux.bin -c isolinux/boot.cat \
  -no-emul-boot -boot-load-size 4 -boot-info-table  \
  -eltorito-alt-boot -e images/efiboot.img -no-emul-boot -graft-points \
  -R -J -v -T -V "KS_RHEL" .

Step 8 - Delete Build Directory

Removes the {ksAttuneBaseDir}/build-{targetServer.fqn} directory.

Execute the following script: 

                
DIR="{ksAttuneBaseDir}/build-{targetServer.fqn}"
if [ -d "${DIR}" ]
then
    rm -rf "${DIR}" && echo "Removed ${DIR}"
else
    echo "No directory to remove."
fi
